# UDA 站点页面规则与操作说明（汇总）

> 更新时间：2026-04-29  
> 说明：本文件汇整本轮沟通中确认并落地的页面规则、字段规则、状态流转与操作约束。

## 一、UDA快件 / 仓内操作（集包模块）

### 1. 菜单与页面命名
- 菜单使用：`UDA快件 / 仓内操作 / 集包列表`、`集包录入`、`集包修改`（原“批次数据”相关命名已统一为“集包”语义）。

### 2. 集包业务主流程
- 流程：`提单号（后改为日期号主键） -> 集包号(auto-001...) -> 扫码面单 -> 填尺寸重量 -> 集包完成 -> 提单完成`。

### 3. 关键字段与唯一性
- `集包号`按提单维度自动顺序编号（如 `260429-001`）。
- `面单号（tracking_no）`全局唯一，不可重复入库。
- 集包主表与明细表均做唯一性与索引约束，避免重复绑定。

### 4. 字段演进规则（重点）
- 原“提单号”主标识改名为`日期号`（界面与数据库同步）。
- 新增独立`提单号`字段，用于显示、搜索、联动，不替代`日期号`主作用。

### 5. 列表/详情/编辑页规则
- 列表支持按`日期号`、`提单号`、`面单号`检索（按页面实际入口提供）。
- 详情与修改页“集包基本信息”字号、字重、颜色已与“日期号 260429”主标题对齐放大。
- 录入与修改页支持滚动位置记忆（返回后尽量回到原操作位置）。

### 6. 交互规则
- 面单扫描校验使用即时弹层/Toast反馈（成功、重复、格式错误等）。
- 提供“放弃提单/放弃集包”类安全退出操作，防止半成品数据污染。

## 二、UDA快件 / 批次操作（新增独立模块）

### 1. 菜单与页面
- 菜单统一为：`UDA快件 / 批次操作 / 批次列表`、`批次录入`、`批次修改`、`批次详情`。
- 旧“仓内操作（下方重复菜单）”已更名，避免与集包模块混淆。

### 2. 批次录入页结构
- 区块1：`日期号`、`提单号`、`UDA件数`、`JD件数`、`总件数`、`航班日期`、`清关完成提货日期`。
- 区块2：CSV导入面单（字段：`面单号`、`日期号`），并与区块1 `日期号`关联。

### 3. 空值与计算规则
- `UDA件数`、`JD件数`、`航班日期`、`清关完成提货日期`允许为空。
- `总件数`自动计算：
  - 两个件数均为空时，`总件数`为空；
  - 任一非空时按数值求和（空按0处理）。

### 4. CSV导入规则（增强版）
- 支持编码：UTF-8（含BOM）、UTF-16LE/BE、GBK/BIG5 自动识别并转UTF-8。
- 支持分隔符自动识别：`,` `;` `Tab` `|`。
- 表头容错：去BOM、空白归一、别名兼容（如 `tracking_no` 映射“面单号”）。
- 提供中文表头模板下载入口，确保可直接导入。

### 5. 批次列表与详情规则
- 列表字段：`日期号`、`提单号`、`UDA件数`、`JD件数`、`总件数`、`航班日期`、`清关完成提货日期`、操作。
- 空值在列表展示为 `-`，导出CSV空值输出为空字符串，避免乱码（已加UTF-8 BOM）。
- 操作：查/改/删；“查”进入明细并可下载该日期号文档。

### 6. 与集包模块联动
- 批次新增/修改后，若`日期号`匹配集包模块记录，自动将`提单号`同步到集包数据中。

## 三、派送业务 / 派送操作（客户维度重构）

### 1. 规则重心
- 派送以“客户”为主，不以“单票订单”为主。
- 某客户派送完成，表示该客户当前范围内相关货件完成派送动作。

### 2. 派送列表
- 展示：`客户编码`、`微信/Line号`、`派送件数（当前已入库数量）`、`泰文小区`、`巷/门牌号`、`主/副线路（升序）`、`定位`。

### 3. 绑带列表
- 展示：`客户编码`、`微信/Line号`、`主/副线路（升序）`、`绑带件数（当前已入库数量）`、`完成按钮`。
- 点击“完成”后，该客户条目从当前绑带列表消失（对应状态流转执行）。

### 4. 生成派送单
- 输入：`派送线(A~E)`、`预计派送日期`。
- 点击“生成派送单号”规则：`YYYYMMDD-线路`。
- 在客户列表勾选后生成派送单：关联单据、更新货件状态并从可派送列表移除。

### 5. 派送单列表
- 展示聚合字段：`派送单号`、`派送线`、`预计派送日期`、`客户数`、`总件数`、`生成时间`。
- 支持“查看明细”：客户编码、联系方式、线路、件数等。

## 四、派送业务 / 转发操作（待转发规则）

### 1. 待转发定义（现行）
- 到件扫描时：
  - OT路线客户 -> 默认进`待转发`；
  - 非OT -> `已入库`。
- 兼容历史数据：候选列表允许读取`待转发`与符合条件的`已入库`数据。

### 2. 手动推送待转发
- 在`派送业务 / 货件操作`提供“手动推送待转发”区域。
- 输入客户编码后查询该客户`已入库`订单，默认全选，可部分勾选。
- 提交后仅将勾选单改为`待转发`。

### 3. 从转发合包回滚
- 在转发合包候选列表可执行“移回已入库”。
- 回滚后该票状态变为`已入库`，并将该票标记`auto_forward_opt_out=1`：
  - 仅限制这张票不再自动进入待转发；
  - 不影响同客户其他票。

### 4. 转发查询记录删除回滚
- 删除某转发包后，包内关联订单回滚为`待转发`（不是`已入库`），便于重新合包。

### 5. 候选查询稳健性
- 转发候选查询已同时兼容按`delivery_customer_id`和`customer_code`匹配，避免因历史数据缺失ID而漏单。

- 全站详情弹窗风格对齐`派送业务 / 转发操作 / 查询记录`详情弹窗样式基线：
  - 遮罩透明度与层级统一；

- `calendar/events` 事件详情弹窗。
