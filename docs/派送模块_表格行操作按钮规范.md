# 派送模块 · 表格行内操作按钮规范（E / D / i）

本文档为 **UDA-V2 派送业务相关列表** 中「行内小操作按钮」的**唯一标准**。之后凡在派送模块（或风格需与派送客户页一致的其他表格）**新增或修改**同类按钮，均应遵循本规范，样式定义在 **`app/Views/layouts/main.php`** 的全局 CSS 中，**勿在页面内重复造一套尺寸/颜色**。

---

## 1. 形态与尺寸

| 项目 | 取值 |
|------|------|
| 外形 | **正圆**（`border-radius: 999px`） |
| 尺寸 | **28×28 px**（`width` / `height` / `min-width` / `min-height` 均为 28px） |
| 内边距 | **0**（圆内仅放单字，不靠 padding 撑宽高） |
| 多按钮排列 | 外层使用 **`dispatch-row-actions`**：`display: inline-flex`，**`gap: 6px`**，不换行优先 `flex-wrap: nowrap` |
| 表格垂直对齐 | 列表 `<table>` 可增加 **`table-valign-middle`**，使圆钮与行内下拉等控件垂直居中 |

---

## 2. 颜色与语义

| 类型 | 文案 | 背景色 | 文字色 | CSS 修饰类 |
|------|------|--------|--------|-------------|
| 编辑 / 主操作 | **E** | `#0b3a82` | `#fff` | `btn-dispatch-round--edit` |
| 详情 / 辅助信息 | **i**（小写） | `#0b3a82`（与 E 一致） | `#fff` | `btn-dispatch-round--info` |
| 删除 | **D** | `#b91c1c` | `#fff` | `btn-dispatch-round--delete` |

悬停时在主样式文件中略加深背景（**勿**使用全局 `.btn:hover` 的亮蓝 `#1d4ed8` 盖掉红/深蓝）。

---

## 3. HTML 结构（复制即用）

```html
<div class="dispatch-row-actions">
    <button type="button" class="btn btn-dispatch-round btn-dispatch-round--edit" title="编辑">E</button>
    <button type="button" class="btn btn-dispatch-round btn-dispatch-round--delete" title="删除">D</button>
    <button type="button" class="btn btn-dispatch-round btn-dispatch-round--info" title="详情">i</button>
</div>
```

- 外层统一使用 **`<div class="dispatch-row-actions">`**（勿用 `<span>`）：当行内包含 **`<form>`**（例如 POST 删除）时，`<form>` 不能合法嵌在 `span` 内，浏览器会改写 DOM，易导致 **D 提交无反应**。
- 若其中一项为 **`<a href="...">`**，同样加上 `btn btn-dispatch-round btn-dispatch-round--edit`（或对应修饰类）。
- **删除**若必须放在 `<form method="post">` 内，仅将 **`<button type="submit">`** 设为上述 `btn-dispatch-round--delete` 即可，**`form` 保持 `display:inline;margin:0`**，与相邻圆钮仍放在同一 `dispatch-row-actions` 内。

---

## 4. 样式源码位置

- **定义**：`app/Views/layouts/main.php` 内 `<style>` 中，紧接在 `.btn-action-forward` 之后的 **`dispatch-row-actions` / `btn-dispatch-round`** 相关规则。
- **参考实现**（已对齐的页面）：
  - `app/Views/dispatch/delivery_customers.php` — 派送客户（基准）
  - `app/Views/dispatch/index.php` — 订单查询
  - `app/Views/dispatch/forwarding/index.php` — 转发客户维护
  - `app/Views/dispatch/consigning_clients.php` — 委托客户

---

## 5. 与全站其它按钮的关系

- **列表行内**的 E / D / i：**必须用** `btn-dispatch-round` 系列，**不要**再用「普通 `.btn` + `padding:4px 10px` 矩形」冒充同款。
- **表单主操作、筛选区、弹窗底部**等仍用现有 **`button` / `.btn`** 规范即可，不必强行改成圆钮。

---

## 6. 修改流程提醒

1. 优先只改 **`main.php`** 中的公共类，实现一次全站同步。  
2. 新页面只拼 **类名 + 结构**，避免内联 `background` / `width` / `border-radius` 与规范不一致。  
3. 若产品上要改直径或色值，**只改文档 §1～§2 与 `main.php` 一处**，并同步更新本文档日期说明。
